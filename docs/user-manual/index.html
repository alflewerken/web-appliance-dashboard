Web-Appliance-Dashboard

Das Schweizer Taschenmesser für die Homelab Administration

Das Web-Appliance-Dashboard ist die zentrale Verwaltungsoberfläche für alle Ihre Homelab-Services. Es vereinfacht die Administration erheblich, indem es als universelles Starterwidget und Aufrufwidget für alle Dienste in Ihrem Netzwerk fungiert.

Keine Lesezeichen mehr nötig! Verwalten Sie alle Ihre Services zentral an einem Ort, ohne sich URLs merken oder Browser-Lesezeichen pflegen zu müssen.

Das Dashboard in der Desktop-Ansicht mit allen wichtigen Services auf einen Blick

Das Dashboard in der mobilen Ansicht

Hauptfunktionen

🚀 Service-Management

Erstellen, bearbeiten und löschen Sie Services mit wenigen Klicks. Jeder Service kann individuell konfiguriert werden.

⚡ Custom Commands

Definieren Sie eigene Befehle für jeden Service, die per Mausklick ausgeführt werden können - ohne Terminal!

📊 Status-Überwachung

Sehen Sie auf einen Blick, welche Services laufen (grün) und welche offline sind (rot).

📝 Audit-Log

Vollständige Protokollierung aller Änderungen mit der Möglichkeit, jeden Zustand wiederherzustellen.

👥 Benutzerverwaltung

Verwalten Sie Benutzer und deren Zugriffsrechte auf verschiedene Services und Funktionen.

💾 Backup & Restore

Sichern Sie Ihre Konfiguration und stellen Sie diese bei Bedarf wieder her.

Vorteil: Alle Änderungen sind reversibel! Versehentlich gelöschte Services können jederzeit wiederhergestellt werden.

Erste Schritte

Installation und Erstkonfiguration

  1. Starten Sie das Dashboard über Ihren Browser: http://localhost:9080
  2. Melden Sie sich mit Ihren Administrator-Zugangsdaten an
    Standard-Anmeldedaten nach der Installation:
    Benutzername: admin
    Passwort: admin123
    Ändern Sie das Passwort nach der ersten Anmeldung!
  3. Im Dashboard sind einige Beispiel-Services angelegt. Löschen Sie diese und drücken Sie dann den "+" Button oben links (neben "Meine Services")
  4. Fügen Sie Ihre ersten eigenen Services hinzu
  5. Konfigurieren Sie Kategorien für eine bessere Organisation

Dashboard-Übersicht

Nach der Anmeldung sehen Sie die Hauptansicht mit allen konfigurierten Services. Jeder Service wird als Karte dargestellt, die wichtige Informationen auf einen Blick zeigt.

Services verwalten

Service-Karten

Jeder Service wird als Karte mit folgenden Informationen dargestellt:

  • Name und Icon: Zur schnellen Identifikation
  • Status-Anzeige: Grün = läuft, Rot = offline
  • URL: Direkter Link zum Service
  • Custom Commands: Schnellzugriff auf häufige Aktionen

Eine Service-Card in der Detailansicht mit grünem Status (Service läuft)

Buttons auf der Service-Card erklärt:

  • 🔗 URL-Button (Hauptbereich): Öffnet die Service-URL in einem neuen Tab
  • ⭐ Favoriten-Stern (oben rechts): Markiert den Service als Favorit für schnelleren Zugriff
  • ✏️ Bearbeiten-Button: Öffnet das Bearbeitungs-Modal für diesen Service
  • ▶️ Start-Button: Führt den konfigurierten Start-Befehl aus
  • ⏹️ Stop-Button: Führt den konfigurierten Stop-Befehl aus
  • 🔄 Status-Button: Aktualisiert den Service-Status manuell
  • 📋 Command-Buttons: Führen die konfigurierten Custom Commands aus
  • 🖥️ Terminal-Button: Öffnet eine SSH-Terminal-Verbindung (wenn konfiguriert)

Eine Service-Card ohne Details mit rotem Status (Service läuft nicht)

Neuen Service hinzufügen

Der Dialog zum Anlegen eines neuen Services

Eingabefelder im Detail:

Basis-Informationen:
  • Name: Der Anzeigename des Services (z.B. "Proxmox", "Home Assistant")
  • URL: Die vollständige URL inklusive Protokoll (z.B. https://192.168.1.100:8006)
  • Beschreibung: Optionale Beschreibung des Services
  • Kategorie: Wählen Sie eine Kategorie zur Organisation (z.B. "Netzwerk", "Media")
  • Icon: Wählen Sie ein passendes Icon aus der Liste
  • Farbe: Akzentfarbe für die Service-Card
URL-Aufruf Einstellungen:
  • In neuem Tab öffnen: Standardmäßig aktiviert - öffnet den Service in einem neuen Browser-Tab
  • Frame-Modus: Zeigt den Service in einem eingebetteten iFrame an (funktioniert nicht bei allen Services)
  • Authentifizierung erforderlich: Markiert Services, die eine Anmeldung benötigen
  • Skip SSL Verification: Nur für selbst-signierte Zertifikate in lokalen Netzwerken
Service-Steuerung (optional):
  • Start-Befehl: Shell-Befehl zum Starten des Services (z.B. docker start container-name)
  • Stop-Befehl: Shell-Befehl zum Stoppen des Services (z.B. docker stop container-name)
  • Status-Befehl: Shell-Befehl zur Status-Abfrage (z.B. docker ps | grep container-name)
SSH-Verbindung (optional):
  • SSH-Profil: Wählen Sie ein konfiguriertes SSH-Profil für Remote-Befehle
  • Terminal aktivieren: Ermöglicht direkten SSH-Zugriff über das Web-Terminal
Erweiterte Optionen:
  • Transparenz: Transparenz-Wert der Service-Card (0-100%)
  • Blur-Effekt: Weichzeichnung des Hintergrunds hinter der Card
  • Als Favorit markieren: Service wird bevorzugt angezeigt
  1. Klicken Sie auf den "+ Service hinzufügen" Button oben links
  2. Füllen Sie mindestens Name und URL aus
  3. Wählen Sie ein passendes Icon und eine Kategorie
  4. Konfigurieren Sie bei Bedarf die Service-Steuerung
  5. Fügen Sie optional Custom Commands hinzu
  6. Klicken Sie auf "Speichern"
Tipp: Sie können Services auch ohne Start/Stop-Befehle anlegen. Diese dienen dann als reine Lesezeichen mit Status-Überwachung.

Service bearbeiten

Um einen bestehenden Service zu bearbeiten:

  1. Klicken Sie auf das Bearbeiten-Symbol (✏️) auf der Service-Card
  2. Das Bearbeitungs-Modal öffnet sich mit allen aktuellen Einstellungen
  3. Nehmen Sie die gewünschten Änderungen vor
  4. Klicken Sie auf "Speichern" um die Änderungen zu übernehmen
Gut zu wissen: Alle Änderungen werden im Audit-Log protokolliert und können bei Bedarf rückgängig gemacht werden.

Service löschen

So entfernen Sie einen Service:

  1. Klicken Sie auf das Bearbeiten-Symbol (✏️) auf der Service-Card
  2. Im Bearbeitungs-Dialog finden Sie unten den "Löschen"-Button
  3. Klicken Sie auf den Löschen-Button
  4. Bestätigen Sie die Sicherheitsabfrage
  5. Der Service wird entfernt, bleibt aber im Audit-Log erhalten
Keine Sorge: Gelöschte Services können jederzeit über das Audit-Log wiederhergestellt werden. Navigieren Sie dazu zu "Einstellungen" → "Audit-Log" und suchen Sie nach der Lösch-Aktion.

Custom Commands

Custom Commands sind einer der mächtigsten Features des Dashboards. Sie ermöglichen es, häufig verwendete Befehle mit einem einzigen Klick auszuführen.

Custom Commands für schnellen Zugriff auf häufige Aktionen

Beispiele für Custom Commands:

  • Docker Container neu starten: docker restart container-name
  • Service-Status prüfen: systemctl status service-name
  • Logs anzeigen: docker logs -f container-name
  • Backup erstellen: ./backup-script.sh
Tipp: Custom Commands werden auf dem Server ausgeführt, auf dem das Dashboard läuft. Stellen Sie sicher, dass alle benötigten Berechtigungen vorhanden sind.

Command erstellen

  1. Öffnen Sie die Service-Bearbeitung
  2. Navigieren Sie zum Abschnitt "Custom Commands"
  3. Klicken Sie auf "Command hinzufügen"
  4. Geben Sie einen aussagekräftigen Namen ein
  5. Fügen Sie den auszuführenden Befehl ein
  6. Optional: Konfigurieren Sie eine Bestätigungsabfrage

Service-Status Überwachung

Das Dashboard überwacht automatisch den Status aller konfigurierten Services:

  • Grüne Statusleiste: Service ist erreichbar und läuft
  • Rote Statusleiste: Service ist nicht erreichbar oder offline
  • Gelbe Statusleiste: Service antwortet langsam oder teilweise

Status-Prüfung konfigurieren

Für jeden Service können Sie individuell festlegen:

  • Prüfintervall (Standard: 60 Sekunden)
  • Timeout für Antworten
  • Erwarteter HTTP-Statuscode
  • Benachrichtigungen bei Statusänderungen

Benutzerverwaltung

Die integrierte Benutzerverwaltung ermöglicht es, verschiedene Zugriffsebenen zu definieren:

Die Benutzerverwaltung mit Übersicht aller angelegten Benutzer

Benutzerrollen

  • Administrator: Vollzugriff auf alle Funktionen
  • Power User: Kann Services verwalten, aber keine Systemeinstellungen ändern
  • Benutzer: Kann nur Services aufrufen, keine Verwaltungsfunktionen
  • Gast: Eingeschränkter Zugriff auf ausgewählte Services

Benutzerdetails bearbeiten und Berechtigungen festlegen

Benutzer anlegen

  1. Navigieren Sie zu "Einstellungen" → "Benutzerverwaltung"
  2. Klicken Sie auf "Neuer Benutzer"
  3. Füllen Sie die Benutzerinformationen aus
  4. Wählen Sie eine Rolle aus
  5. Optional: Beschränken Sie den Zugriff auf bestimmte Service-Kategorien

Audit-Log

Das Audit-Log ist ein zentrales Sicherheitsfeature des Dashboards. Hier werden alle Änderungen protokolliert und können bei Bedarf rückgängig gemacht werden.

Das Audit-Log zeigt alle Änderungen chronologisch an

Protokollierte Aktionen

  • Service erstellt/geändert/gelöscht
  • Benutzer angelegt/bearbeitet/entfernt
  • Systemeinstellungen geändert
  • Custom Commands ausgeführt
  • Backup/Restore durchgeführt
  • SSH-Hosts hinzugefügt/geändert/entfernt (NEU in v1.0.3)
  • SSH-Keys verwaltet
Neu in Version 1.0.3: Das Audit-Log unterstützt jetzt auch Live-Updates für SSH-Host Änderungen. Wenn ein anderer Administrator SSH-Hosts ändert, sehen Sie die Änderungen sofort im Audit-Log, ohne die Seite neu laden zu müssen.

Filteroptionen für verschiedene Aktionstypen

Mit dem Audit-Log arbeiten

Einträge filtern

Sie können das Audit-Log nach verschiedenen Kriterien filtern:

Filtern nach Ressourcen-Typ

  • Zeitraum: Heute, letzte Woche, letzter Monat, benutzerdefiniert
  • Benutzer: Aktionen eines bestimmten Benutzers
  • Aktionstyp: Erstellt, geändert, gelöscht
  • Ressource: Services, Benutzer, Einstellungen

Änderungen rückgängig machen

Detailansicht einer Service-Änderung mit Möglichkeit zur Wiederherstellung

  1. Suchen Sie den entsprechenden Eintrag im Audit-Log
  2. Klicken Sie auf "Details anzeigen"
  3. Überprüfen Sie die Änderungen
  4. Klicken Sie auf "Wiederherstellen" um zum vorherigen Zustand zurückzukehren
Achtung: Das Wiederherstellen einer Änderung wird ebenfalls im Audit-Log protokolliert und kann selbst wieder rückgängig gemacht werden.

Backup & Restore

Sichern Sie regelmäßig Ihre Dashboard-Konfiguration, um im Notfall schnell wiederherstellen zu können.

Backup erstellen und Wiederherstellung durchführen

Was wird gesichert?

  • Alle Service-Konfigurationen
  • Benutzerdaten und Berechtigungen (inkl. Password-Hashes seit v1.0.2)
  • Systemeinstellungen
  • Kategorien und Tags
  • Custom Commands
  • Dashboard-Layout Einstellungen
  • SSH-Hosts und SSH-Keys
  • Hintergrundbilder (als Base64)
  • Audit-Log Historie
Wichtig seit Version 1.0.2: Backups enthalten jetzt auch Benutzer-Passwort-Hashes für eine vollständige Systemwiederherstellung. Bewahren Sie Backup-Dateien sicher auf, da sie sensitive Daten enthalten.

Backup erstellen

  1. Navigieren Sie zu "Einstellungen" → "Backup & Restore"
  2. Klicken Sie auf "Backup erstellen"
  3. Wählen Sie optional aus, was gesichert werden soll
  4. Das Backup wird als ZIP-Datei heruntergeladen

Backup wiederherstellen

  1. Navigieren Sie zu "Einstellungen" → "Backup & Restore"
  2. Klicken Sie auf "Backup hochladen"
  3. Wählen Sie die Backup-Datei aus
  4. Überprüfen Sie die Vorschau der wiederherzustellenden Daten
  5. Bestätigen Sie die Wiederherstellung
Tipp: Erstellen Sie automatische Backups über einen Cron-Job für maximale Sicherheit.

Kategorien verwalten

Organisieren Sie Ihre Services in Kategorien für eine bessere Übersicht.

Kategorien erstellen und verwalten

Vorteile von Kategorien

  • Gruppierung ähnlicher Services
  • Schnellere Navigation bei vielen Services
  • Berechtigungen pro Kategorie vergeben
  • Farbliche Kennzeichnung möglich

Kategorie erstellen

  1. Gehen Sie zu "Einstellungen" → "Kategorien"
  2. Klicken Sie auf "Neue Kategorie"
  3. Vergeben Sie einen Namen
  4. Wählen Sie optional eine Farbe und ein Icon
  5. Definieren Sie die Sortierreihenfolge

Beispielkategorien

  • Netzwerk: Router, Switches, Firewall
  • Media: Plex, Jellyfin, Sonarr, Radarr
  • Verwaltung: Proxmox, Portainer, phpMyAdmin
  • Monitoring: Grafana, Prometheus, Uptime Kuma
  • Entwicklung: GitLab, Jenkins, Code-Server

Hintergrundbild anpassen

Personalisieren Sie Ihr Dashboard mit einem eigenen Hintergrundbild.

Hintergrundbild auswählen und anpassen

Einstellungsmöglichkeiten

  • Bild hochladen: JPG, PNG oder WebP bis 5MB
  • Transparenz: Overlay-Stärke für bessere Lesbarkeit
  • Blur-Effekt: Weichzeichnung des Hintergrunds
  • Positionierung: Zentriert, gestreckt oder gekachelt

Empfehlungen

  • Verwenden Sie Bilder mit mindestens 1920x1080 Pixeln
  • Dunkle Bilder funktionieren oft besser mit dem UI
  • Abstrakte oder unscharfe Bilder lenken weniger ab
  • Testen Sie verschiedene Transparenz-Einstellungen

SSH Remote Control

Führen Sie Commands nicht nur lokal, sondern auch auf entfernten Servern aus.

Verbessert in Version 1.0.3: SSH-Host Verwaltung mit vollständiger Audit-Log Integration:
  • Live-Updates bei SSH-Host Änderungen
  • Wiederherstellung von SSH-Host Konfigurationen über das Audit-Log
  • Detaillierte Änderungshistorie mit Vorher/Nachher-Vergleich

SSH-Verbindungen für Remote-Commands konfigurieren

Features

  • Sichere SSH-Verbindungen zu Remote-Servern
  • Ausführung von Commands auf entfernten Systemen
  • SSH-Key basierte Authentifizierung
  • Mehrere SSH-Profile verwalten

SSH-Profil einrichten

  1. Navigieren Sie zu "Einstellungen" → "SSH Remote Control"
  2. Klicken Sie auf "Neues SSH-Profil"
  3. Geben Sie Hostname/IP und Port ein
  4. Wählen Sie die Authentifizierungsmethode
  5. Testen Sie die Verbindung
  6. Speichern Sie das Profil

Remote Commands verwenden

Bei der Erstellung von Custom Commands können Sie nun wählen:

  • Lokal: Command wird auf dem Dashboard-Server ausgeführt
  • Remote: Command wird über SSH auf dem Zielserver ausgeführt
Sicherheit: Verwenden Sie ausschließlich SSH-Keys für die Authentifizierung. Passwort-basierte Authentifizierung sollte vermieden werden.

Tipps & Tricks

Performance-Optimierung

  • Status-Checks: Erhöhen Sie das Prüfintervall für weniger wichtige Services
  • Lazy Loading: Aktivieren Sie verzögertes Laden für große Service-Listen
  • Caching: Nutzen Sie den Browser-Cache für statische Ressourcen

Sicherheit

  • HTTPS: Verwenden Sie immer verschlüsselte Verbindungen
  • 2FA: Aktivieren Sie Zwei-Faktor-Authentifizierung für Administratoren
  • API-Keys: Rotieren Sie regelmäßig API-Schlüssel
  • Audit-Log: Überprüfen Sie regelmäßig verdächtige Aktivitäten

Workflow-Verbesserungen

  • Tastenkürzel:
    • Strg/Cmd + K: Schnellsuche
    • Strg/Cmd + N: Neuer Service
    • Strg/Cmd + S: Speichern
    • ESC: Dialog schließen
  • Favoriten: Markieren Sie häufig genutzte Services als Favoriten
  • Tags: Verwenden Sie Tags zusätzlich zu Kategorien für flexiblere Organisation

Integration

  • Webhooks: Lassen Sie sich bei Service-Ausfällen benachrichtigen
  • API: Nutzen Sie die REST-API für Automatisierungen
  • Monitoring: Integrieren Sie externe Monitoring-Tools
Pro-Tipp: Erstellen Sie Service-Templates für häufig verwendete Konfigurationen, um neue Services schneller anzulegen.

Neue Features in Version 1.0.3

Die aktuelle Version 1.0.3 bringt wichtige Verbesserungen für die SSH-Host Verwaltung:

SSH-Host Audit-Log Funktionalität

  • Live-Updates: Änderungen an SSH-Hosts werden in Echtzeit im Audit-Log angezeigt
  • Wiederherstellung: SSH-Host Konfigurationen können über das Audit-Log wiederhergestellt werden
  • Detaillierte Visualisierung: Vorher/Nachher-Vergleich bei SSH-Host Änderungen
  • Verbesserte Stabilität: Behebung von Race Conditions und 404-Fehlern

Technische Verbesserungen

  • Erweiterte SSE (Server-Sent Events) für SSH-Host Events
  • Verbesserte Live-Updates im Einstellungen-Panel
  • Optimierte Fehlerbehandlung bei Wiederherstellungsvorgängen
  • Backup-Version auf 2.8.0 aktualisiert
Upgrade-Hinweis: Beim Update von Version 1.0.2 auf 1.0.3 wird empfohlen, vor dem Update ein vollständiges Backup zu erstellen. Die neue Version ist vollständig abwärtskompatibel.
