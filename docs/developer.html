Installation

Systemvoraussetzungen

Das Web Appliance Dashboard läuft in Docker-Containern und benötigt folgende Voraussetzungen:

  • Docker Desktop (Windows/Mac) oder Docker Engine (Linux)
  • Docker Compose v2.0 oder höher
  • Mindestens 2 GB freier RAM
  • Mindestens 1 GB freier Festplattenspeicher
  • Internetzugang für das Herunterladen der Docker-Images

🐳 Docker Installation

Windows Docker Desktop für Windows

Installation:

  1. Laden Sie Docker Desktop herunter
  2. Führen Sie den Installer aus und folgen Sie den Anweisungen
  3. Starten Sie Docker Desktop nach der Installation
  4. Öffnen Sie PowerShell oder Command Prompt und prüfen Sie die Installation:
docker --version
docker compose version

Systemanforderungen:

  • Windows 10 64-bit: Pro, Enterprise, oder Education (Build 19041 oder höher)
  • Windows 11 64-bit: Home, Pro, Enterprise, oder Education
  • WSL 2 Backend aktiviert (wird automatisch installiert)
  • Virtualisierung im BIOS aktiviert
Hinweis: Bei Windows Home Edition muss WSL 2 verwendet werden. Docker Desktop installiert dies automatisch.
macOS Docker Desktop für Mac

Installation:

  1. Laden Sie Docker Desktop für Mac herunter
  2. Doppelklicken Sie auf Docker.dmg und ziehen Sie Docker in den Applications-Ordner
  3. Starten Sie Docker aus dem Applications-Ordner
  4. Öffnen Sie Terminal und prüfen Sie die Installation:
docker --version
docker compose version

Systemanforderungen:

  • macOS 10.15 (Catalina) oder neuer
  • Apple Silicon (M1/M2/M3) oder Intel-Prozessor
  • Mindestens 4 GB RAM (8 GB empfohlen)

Installation via Homebrew (alternativ):

brew install --cask docker
Linux Docker Engine für Linux

Installation für Ubuntu/Debian:

# Alte Versionen entfernen
sudo apt-get remove docker docker-engine docker.io containerd runc

# Repository-Setup
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg lsb-release

# Docker GPG-Schlüssel hinzufügen
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# Repository hinzufügen
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Docker installieren
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Benutzer zur docker-Gruppe hinzufügen
sudo usermod -aG docker $USER
newgrp docker

# Installation prüfen
docker --version
docker compose version

Installation für Fedora/CentOS/RHEL:

# Repository-Setup
sudo dnf -y install dnf-plugins-core
sudo dnf config-manager --add-repo https://download.docker.com/linux/fedora/docker-ce.repo

Installation für Arch Linux:

⚙️ Konfiguration

Ports Netzwerk-Konfiguration

Standard-Ports:

# docker-compose.yml anpassen
services:
  webserver:
    ports:
      - "9080:80"    # Frontend auf Port 9080
  
  backend:
    ports:
      - "3001:3001"  # Backend API auf Port 3001
  
  database:
    ports:
      - "3306:3306"  # MariaDB (optional, für externe Zugriffe)
  
  ttyd:
    ports:
      - "7681:7681"  # Web Terminal

Ports ändern:

Falls die Standard-Ports bereits belegt sind, können Sie diese in der docker-compose.yml ändern:

# Beispiel: Frontend auf Port 8000 ändern
services:
  webserver:
    ports:
      - "8000:80"    # Geändert von 9080 auf 8000
Hinweis: Nach Port-Änderungen müssen Sie den webserver Container neu starten: docker compose restart webserver
Volumes Datenpersistenz

Docker Volumes:

Die Anwendung verwendet Docker Volumes für persistente Datenspeicherung:

volumes:
  db_data:         # MariaDB Datenbank-Dateien
  ssh_keys:        # SSH-Schlüssel für Terminal-Zugriff

Backup erstellen:

# MariaDB Datenbank exportieren
docker compose exec database mysqldump -u root -p appliance_dashboard > backup.sql

# Volumes sichern
docker run --rm -v web-appliance-dashboard_db_data:/data -v $(pwd):/backup ubuntu tar czf /backup/db_data_backup.tar.gz /data

Backup wiederherstellen:

# MariaDB Datenbank importieren
docker compose exec -T database mysql -u root -p appliance_dashboard < backup.sql

# Volumes wiederherstellen
docker run --rm -v web-appliance-dashboard_db_data:/data -v $(pwd):/backup ubuntu tar xzf /backup/db_data_backup.tar.gz -C /

🔧 Fehlerbehebung

Häufige Probleme Lösungsansätze

1. Docker Daemon läuft nicht:

# Windows/Mac: Docker Desktop starten
# Linux: Docker Service starten
sudo systemctl start docker

# Status prüfen
docker info

2. Port bereits belegt:

# Belegte Ports finden
# Windows
netstat -ano | findstr :9080

# Mac/Linux
lsof -i :9080

# Lösung: Ports in docker-compose.yml ändern oder belegenden Prozess beenden

3. Container startet nicht:

# Logs prüfen
docker compose logs backend
docker compose logs webserver
docker compose logs database
docker compose logs ttyd

# Container neu starten
docker compose down
docker compose up -d

# Mit sauberen Volumes neu starten
docker compose down -v
docker compose up -d

4. Verbindungsprobleme Frontend zu Backend:

# Nginx-Konfiguration prüfen
docker compose exec webserver cat /etc/nginx/conf.d/default.conf

# Container neu bauen nach Konfigurationsänderungen
docker compose restart webserver

5. MariaDB Verbindungsfehler:

# MariaDB Container prüfen
docker compose exec database mysql -u root -p

# Bei Passwort-Problemen: Volume löschen und neu starten
docker compose down -v
docker compose up -d

6. Speicherplatz-Probleme:

# Docker Speicherplatz aufräumen
docker system prune -a --volumes

# Nur ungenutzte Images löschen
docker image prune -a

# Container-Logs begrenzen
docker compose logs --tail=100
Debug-Modus: Für detaillierte Fehleranalyse können Sie den Debug-Modus in der .env-Datei aktivieren: NODE_ENV=development

docker compose logs

docker compose logs backend -f

docker compose ps

docker stats

# Alle Container stoppen

Bekannte Probleme:

  • Windows + WSL2: Langsame Performance bei Dateien außerhalb des WSL-Dateisystems
  • Mac M1/M2: Manche Images benötigen Platform-Flag: --platform linux/amd64
  • Firewall: Ports müssen in der Firewall freigegeben sein
  • Antivirus: Kann Docker-Operationen blockieren - Ausnahmen hinzufügen
Tipp: Bei persistenten Problemen hilft oft ein kompletter Neustart: docker compose down && docker compose up -d

🔐 Authentication API

POST /api/auth/login Public

Beschreibung:

Authentifiziert einen Benutzer und gibt ein JWT-Token zurück.

Request Body:

{
  "username": "admin",
  "password": "secretpassword"
}

Response (200 OK):

{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "user": {
    "id": 1,
    "username": "admin",
    "email": "admin@example.com",
    "role": "admin"
  }
}

Error Response (401):

{
  "error": "Invalid credentials"
}

curl Beispiel:

curl -X POST http://localhost:3001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "secretpassword"
  }'

Python Beispiel:

import requests

url = "http://localhost:3001/api/auth/login"
data = {
    "username": "admin",
    "password": "secretpassword"
}

response = requests.post(url, json=data)
if response.status_code == 200:
    result = response.json()
    token = result["token"]
    print(f"Token: {token}")
else:
    print(f"Error: {response.json()}")

curl Beispiel:

curl -X GET http://localhost:3001/api/backup \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." \
  -o backup.json

Python Beispiel:

import requests
import json
from datetime import datetime

url = "http://localhost:3001/api/backup"
headers = {
    "Authorization": "Bearer eyJhbGciOiJIUzI1NiIs..."
}

response = requests.get(url, headers=headers)
if response.status_code == 200:
    backup_data = response.json()
    filename = f"backup_{datetime.now().strftime('%Y%m%d_%H%M%S')}.json"
    
    with open(filename, 'w') as f:
        json.dump(backup_data, f, indent=2)
    
    print(f"Backup saved to {filename}")
    print(f"Version: {backup_data['version']}")
    print(f"Appliances: {len(backup_data['appliances'])}")
else:
    print(f"Error: {response.json()}")

JavaScript/WebSocket Beispiel:

const token = "eyJhbGciOiJIUzI1NiIs...";
const ws = new WebSocket(`ws://localhost:3001/api/terminal/connect?token=${token}`);

ws.onopen = () => {
    console.log('Terminal connected');
    
    // Send command
    ws.send(JSON.stringify({
        type: 'input',
        data: 'ls -la\n'
    }));
};

ws.onmessage = (event) => {
    const message = JSON.parse(event.data);
    if (message.type === 'output') {
        console.log('Terminal output:', message.data);
    }
};

ws.onerror = (error) => {
    console.error('WebSocket error:', error);
};

Python WebSocket Beispiel:

import websocket
import json

token = "eyJhbGciOiJIUzI1NiIs..."
url = f"ws://localhost:3001/api/terminal/connect?token={token}"

def on_message(ws, message):
    data = json.loads(message)
    if data['type'] == 'output':
        print(data['data'], end='')

def on_open(ws):
    # Send a command
    ws.send(json.dumps({
        'type': 'input',
        'data': 'ls -la\n'
    }))

ws = websocket.WebSocketApp(url,
                            on_message=on_message,
                            on_open=on_open)
ws.run_forever()

import sseclient
import requests
import json

token = "eyJhbGciOiJIUzI1NiIs..."

    "Authorization": f"Bearer {token}"
}

# Create SSE client
response = requests.get(url, headers=headers, stream=True)
client = sseclient.SSEClient(response)

# Listen for events
for event in client.events():
    if event.event == 'service_status_changed':
        data = json.loads(event.data)
        print(f"Service {data['name']} is now {data['status']}")
    
    elif event.event == 'appliance_updated':
        data = json.loads(event.data)
        print(f"Appliance {data['id']} updated")
    
    elif event.event == 'appliance_created':
        data = json.loads(event.data)
        print(f"New appliance: {data['name']}")
    
    elif event.event == 'category_updated':
        data = json.loads(event.data)
        print('Categories updated')
    
    elif event.event == 'notification':
        data = json.loads(event.data)
        print(f"{data['type']}: {data['message']}")
